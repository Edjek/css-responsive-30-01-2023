Un Reset CSS c'est quoi ? Pour quoi ?

Les navigateurs web sont tenus d'appliquer des styles par défaut (User Agent Stylesheet) à chaque page HTML, sinon le document afficherait une page blanche.

Chaque navigateur applique ses propres styles par défaut pouvant se révéler différents les uns des autres, sinon ce serait trop facile pour nous autres développeuses et développeurs !

Deux méthodes permettent de contourner ces différences d'affichages entre les navigateurs : le Reset CSS et le Normalize CSS.

Reset et Normalize. Ces outils permettent de contrebalancer le problème des styles par défaut imposés par les navigateurs (par exemple, les paragraphes ont une marge par défaut, marges qui ne sont pas forcément constantes entre les navigateurs).

Un "Reset CSS" (réinitialisation) est une technique qui consiste à repasser toutes les valeurs des propriétés CSS à leur état initial afin repartir de zéro et d'une base vierge avant d'appliquer nos propres styles.

La plus célèbre est le reset d’ Eric Meyer.

Un "Normalize CSS" (harmonisation) consiste à appliquer des styles de base cohérents identiques sur chaque navigateur.
Cette méthode corrige également les différences d'interprétation des spécifications et d'affichage sur l'ensemble des navigateurs et produit une base de travail suffisante (typographie, interlignages, marges, etc.)

Normalize.css, développé par Nicolas Gallagher et Jonathan Neal, est le projet le plus récent et il est mis régulièrement à jour.
Sachez que Normalize.css est utilisé par la plupart des projets de frameworks CSS de développement de site web : Bootstrap, Foundation, Pure, HTML5-Boilerplate...
Normalize.css est "juste" une feuille de style CSS qui harmonise le rendu initial des navigateurs et qui corrige certains bugs.
